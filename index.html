Projet : Extension d'une Architecture sous Debian 13

Contexte

Conception d’un sous-réseau sécurisé pour héberger Grafana sur une VM Debian 13. Intégration dans un réseau existant (172.10.0.0/16) avec des contraintes strictes :

  • Accès SSH uniquement depuis le poste administrateur (172.10.16.254).
  • Pare-feu applicatif pour bloquer/autoriser les accès depuis SW2.
  • Protection contre les attaques par force brute (bannissement 24h après 10 échecs en 2h).
  • Respect du FHS et des bonnes pratiques Debian.

Topologie et Adressage

  • Zone Office (SW1) : 172.10.0.0/20 (3045 postes).
  • Zone Technique (SW2) : 172.10.16.0/24 (200 postes).
  • Nouveau sous-réseau (Grafana) : 172.10.17.0/25 (65 postes).
    • Routeur R1 (Fa1/0) : 172.10.17.1.
    • VM Survey_03.lan : 172.10.17.2 (IP fixe).

Ports ouverts : 22 (SSH), 3000 (Grafana), 80 (HTTP), 443 (HTTPS).

Configuration Clé

Routeur Cisco 3725 :

interface fastEthernet 1/0
 ip address 172.10.17.1 255.255.255.128
 no shutdown

VM Debian :

# Installation Grafana
sudo apt install -y grafana

# Sécurisation SSH
PermitRootLogin no
AllowUsers viz_admin

# Protection (fail2ban)
bantime = 86400
maxretry = 10

Outils Utilisés

  • GNS3 : Simulation du réseau.
  • Debian 13 : VM hébergeant Grafana.
  • fail2ban : Protection contre les attaques.
